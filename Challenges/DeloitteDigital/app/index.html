Answer:

CSS

  • Describe floats and how they work.
    • Float is a CSS property applied when an element should be taken out its determined structure and placed along the left or right edge of its parent element. If an element is floated, other text and inline elements will wrap around it.
  • What is the box model? Create a simple example using a div. Follow up: Where does the border go in the box model?
    • Imagine that each element on your page is a rectangle. The box model describes the different spaces within the element; there are four components: the element itself; the padding, the border, and the margin.
    • It's important to note that the border comes between the outermost space (margin) and the padding, not after the content of the element itself.
  • What does * { box-sizing: border-box; } do? What are its advantages?
    • * { box-sizing: border-box; } specifies that the element's border should be automatically incorporated when calculating the element's width. All browsers use the following method to determine the elements' height/widths: "width or height + padding + border = actual width or height". With box-sizing: border-box;, an element's specified width and height aren't affected by padding or borders.

  • What's the difference between inline and inline-block?
    • Inline is the default value of the display property and it displays elements as inline. (An example of an inline element is a span or a) Inline-block, however, displays an element as an inline level container while the content inside is formatted as block.

JavaScript

  • Explain how this works in JavaScript.
    • this is a special variable that gets assigned based on the exceuction context/scope of a function. In the initial, global context, this evaluates to the global object window. If you call a function via a click event, this gets assigned to the subject of the code that is being executed, or whatever instantiated the function.
  • Explain "hoisting".
    • "Hoisting" is when a variable or function declaration evaluates before any other code is executed. Declaring a variable anywhere in your document is the same as if you had declare it at the top. The catch is that variable assignments do not get hoisted! For example: 
      
// Outputs: undefined
console.log(declaredLater);

var declaredLater = "Now it's defined!";

// Outputs: "Now it's defined!"
console.log(declaredLater);
  • Explain why the following doesn't work as an IIFE: function foo(){ }();. What needs to be changed in order to make it work?
    • The code doesn't work because a function declaration cannot be immediately invoked by a set of parenthesis. However, a function expression can be! If you wrap the function inside an additional set of parentheses, the JavaScript parser will treat the code as a function expression instead of a function declaration.To be an IIFE (Immediately Invoked Function Expression), you need to add a set of parenthesis around the entire function (before the second set). In this case, the code should be changed to: (function foo(){ })();

alert(foo + bar);
